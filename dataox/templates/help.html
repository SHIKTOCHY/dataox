What is Open Data?

Open Data is the idea that data across the web should be published as freely and as openly as is possible to enable others to reuse and republish as they wish.

data.ox.ac.uk is the University of Oxford's hub for Open Data. Currently departments often publish data on their own websites independently- the aim of data.ox.ac.uk is to centralise this (and other) data and to release it to a Five-Star standard!

What is RDF?

RDF is a data model in which facts are represented as a set of triples, each comprising a subject, predicate (verb), and object. For example,

Alice has-age 42
Alice lives-in Oxford

However, the above is a simplification. To make this Five-Star data we need to use URIs and link to other people's data. Therefore, each subject/predicate/object in the above should have a URI (a 'Uniform Resource Identifier'- like a URL, except there doesn't have to be a file at the end of it). For example,

<http://example.com/employees/Alice> <http://example.com/predicates/lives-in> <http://example.com/cities/Oxford>

This raises the grade of of dataset to Four Stars. However, we still haven't linked to other people's data! We should reuse other people's data to provide context to our data; eg to let others know whether our Oxford is the same as someone else's Oxford. For example:

<http://example.com/employees/Alice> <http://xmlns.com/foaf/0.1/based_near> <http://live.dbpedia.org/page/Oxford>

Here, we have used DBpedia's URI for Oxford instead of ours. We have also taken our "based_near" predicate from the FOAF vocabulary. Predicates should (almost) always be drawn from commonly-understood vocabularies, in this case the FOAF vocabulary.

Data represented in RDF can be written down (serialised) in a variety of ways. The most common way is using XML which is how most of data.ox.ac.uk's data is stored. Other ways include Notation3 and Turtle.

What is SPARQL?

SPARQL is a query language for RDF, just as SQL is a query language for relational databases. Each query works by matching graph patterns to data held by data.ox.ac.uk. For example, if we run this query:

PREFIX foaf: <http://xmlns.com/foaf/0.1/>
SELECT ?person WHERE
{
?person foaf:based_near <http://live.dbpedia.org/page/Oxford>
}

on our dataset from "What is RDF?", then <http://example.com/employees/Alice> will be returned to us.

If you are new to SPARQL and wish to make complicated queries to data.ox.ac.uk in SPARQL then you it may be best to read external resources such as Wikipedia. You can also consult out "Example Queries" page, found here.

What is this "βήτα" thing?

data.ox.ac.uk is currently offered on a best-efforts basis as a beta service. This means that we'll do our best to keep it up and running, but make no guarantees.

We hope that data.ox.ac.uk will become a supported service in the not-too-distant future. Watch this space!

What do the stars mean?

The stars represent the extent to which a dataset is part of the linked open data web. These were the definitions used by Tim Berners-Lee during his Gov 2.0 talk in 2010, and have since become universally accepted.

One Star - Put your data on the web (in any format)
Two Stars - Make it available in a machine readable format (i.e. no image scans!)
Three Stars - Use a open, standard format (eg JSON)
Four Stars - Use an open, linked data format (URIs for everything so people can point to your data)
Five Stars - Link your data to other people’s data

We aim to release all data on data.ox.ac.uk to a Five-Star standard. However, it can be a long process to convert data into this format so we may release data to a lower standard in the hope of raising it in the future.

How do I access the data?

The easiest way to access the raw data stored in data.ox.ac.uk is through our SPARQL Endpoint, available here. This can output the data in a variety of formats (see below). However, we have also created a variety of Apps (available here) to visualise this data and make accessing it even easier!

What formats do you publish data in?

Our SPARQL endpoint can output to a variety of formats. For example:

  • RDF/XML
  • Notation3/N3
  • JSON
  • HTML
  • CSV
  • XLS
  • RSS/Atom
  • KML

How do I contribute a dataset?

We would be very happy to incorporate your datasets into data.ox.ac.uk. Please contact us on opendata@oucs.ox.ac.uk.
